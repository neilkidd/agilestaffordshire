Brooklyn Beta<\/strong><\/p>\n

A big thank you \u00a0to Trevor for sharing his brilliant talk on experiences at Brooklyn Beta earlier this year, which focused on web platforms delivering. It is really encouraging to see the move towards organisations exposing data allowing the creation of business that can deliver the real value of that data by matching the needs of those in the community. The event is highly recommended with further details on Trevor’s Blog<\/a>, so if your interested in attending next year, watch out for details here \u00a0–\u00a0https:\/\/brooklynbeta.org<\/a>\/<\/p>\n

Maintainable C<\/strong><\/p>\n

<\/strong>We then moved onto Trevor’s C code used for\u00a0stenography and discussed how it can be tested. The ideas presented for testing involved simple regression tests to ensure it was functionally working using shell scripts, then work on breaking down the single function into logical methods\/applications that are effectively unit tested. You can contribute to the project at Trevor’s repository<\/a>.<\/p>\n

 <\/p>\n

I’ll be sure to get a link up for the slides from Trevor shortly.<\/p>\n

It was good to see new faces, and thanks to everyone for attending.<\/p>\n
