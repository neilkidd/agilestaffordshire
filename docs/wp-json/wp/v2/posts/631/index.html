March is all about code kata. We shall be visiting a kata that has been run before – Anagram! If you do not know what code kata is, start with Dave’s Blog<\/a>\u00a0and then read his anagram kata<\/a>\u00a0post.<\/p>\n

We like change, so consider throwing some into this exercise (particularly those who have done it before). Some suggestions:<\/p>\n

    \n
  • Profile the application, perhaps in both time (execution speed) and space (memory usage).<\/li>\n
  • Consider intermediate list processing and storage (structure the word list).<\/li>\n
  • Try TDD<\/abbr> in a pair.<\/li>\n
  • Complete Dave’s suggestions –\u00a0For added programming pleasure, find the longest words that are anagrams, and find the set of anagrams containing the most words (so \u201cparsley players replays sparely\u201d would not win, having only four words in the set).<\/li>\n<\/ul>\n

    The word list<\/a> we used last time is still available on my blog<\/a>.\u00a0Bring your own laptops and whatever programming tools you wish. Wireless Internet access and a desktop computer with which to cut code will be available should you require it.<\/p>\n
