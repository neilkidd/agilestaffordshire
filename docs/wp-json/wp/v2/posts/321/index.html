Introduction To Kanban<\/h2>\n

At last months meeting<\/a>, due to technical problems, we didn’t get round to the Minisculus Challenge<\/a>. However we did discuss the idea of extending the Alien Invasion project.<\/p>\n

The catalyst of this project is that back in\u00a0 June<\/a> one of the ideas was that we should work on a project together. When we first tried the Alien Invasion<\/a> we found that, due to a dependency on a client side library, participants were constrained to a .net<\/a> implementation. In the discussion following there was a consensus that it would have been better for the clients to be language agnostic.<\/p>\n

The intended outcomes of the project are:<\/p>\n

    \n
  • To refactor the code so that it can be used via any web enabled language.<\/li>\n
  • To act as the catalyst for discussions at our monthly sessions<\/li>\n<\/ul>\n

    It was agreed that Kanban would probably be a good fit for our project management needs, as it doesn’t specify fixed iterations. Additionally a number of the group, while being aware, have not used Kanban “in anger” so it was decided that this months session would be an introduction Kanban.<\/p>\n

    By happy coincidence Ash Moran<\/a> is running a getKanban<\/a> session at madlab in Manchester<\/a> on the 24th<\/sup> November.\u00a0 I’m very pleased to say that Ash has agreed to come in and run the “short” game play session for us. I will be going to the Saturday MadLab session and at the time time of writing there are still places available<\/a><\/strong>, join us if you can.<\/p>\n

     <\/p>\n

    Hope to see you there<\/p>\n
