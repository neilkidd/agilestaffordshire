Date: Wednesday 26 September 2012<\/p>\n

Time: 19:30 start.<\/p>\n

Venue: The Stafford Ale House<\/a>
\nLocation: Google Map<\/a><\/p>\n

It’s a follow up session! Bring your laptops and other programming apparatus. Last month<\/a> we had a great session with the Anagram Kata. This month, we shall follow up with some interesting turns involving the same Kata. We shall assume the roles of developers that have happened across some code regarding Kata and have acquired responsibility of maintenance.<\/p>\n

Trade your solutions from last month for a piece of code that you have not seen. There’s plenty of implementations in various languages in the comments section of our August post<\/a> (Objective-C, Java, C#, Python and PHP amongst others). If you have attempted the kata then I encourage you to share<\/a> your<\/a> code<\/a> for some variety. Adapt the code using one or more of the following scenarios:<\/p>\n

    \n
  1. The code is used in a neat widget in an unspecified number of ways. You’ve been tasked with making the code extensible and adaptable to change. Refactor an existing code to achieve the goal.<\/li>\n
  2. Adapt the algorithm employed in the existing code to perform ‘better’ in terms of time and\/or space. How will you measure this? Can you cache results to improve anagram generation time for subsequent requests of identical input?<\/li>\n
  3. If your acquired code has no formal testing applied to it, how can you be sure that it works? Create a suite of tests. What tests are required to assure required functionality in any dictionary?<\/li>\n<\/ol>\n


    \nThis kata is a nice opportunity to try some things that we don’t always have the opportunity to do. It is also possible to apply various techniques in producing code for any of the scenarios listed (pair programming, TDD<\/abbr> etc.). We will wrap the evening up with some ‘show and tell’ sessions where we can discuss each other’s offerings.<\/p>\n
